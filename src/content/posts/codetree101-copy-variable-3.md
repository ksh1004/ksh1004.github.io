## 핵심 개념

### 변수 간 값 복사

`a = c`는 `c`가 저장하고 있는 **값**을 그대로 `a`에 복사하는 것이다. 이후 `a`를 바꿔도 `c`에는 영향이 없다 — 정수·문자열 같은 값 타입은 독립적으로 복사된다.

```python
c = 3
a = c   # c의 값(3)을 a에 복사
a = 10  # a만 바뀜, c는 여전히 3
```

### 순차 실행과 값 덮어쓰기

코드는 위에서 아래로 한 줄씩 실행되며, 대입이 일어난 순간 **그 줄 이전의 값은 사라진다.** 뒤에 나오는 연산은 항상 최신 값을 기준으로 계산된다.

---

## 접근 방식

한 줄씩 상태를 추적하면 헷갈리지 않는다.

| 실행 줄 | a | b | c |
|---|---|---|---|
| `a, b, c = 1, 5, 3` | 1 | 5 | 3 |
| `a = c` | 3 | 5 | 3 |
| `a = a + c` | 6 | 5 | 3 |
| `b = b - c` | 6 | 2 | 3 |

핵심은 `a = c` 시점에서 **원래 a에 있던 1은 완전히 사라진다**는 것이다. 이후 `a = a + c`는 새로 복사된 `a`(=3)와 `c`(=3)를 더한 값이지, 원래 `a`였던 1과는 무관하다.

---

## 풀이 코드

```python
a, b, c = 1, 5, 3

a = c          # c의 값(3)을 a에 복사 → a는 이제 3
a = a + c      # 새로운 a(3) + c(3) = 6
b = b - c      # b(5) - c(3) = 2

print(a)  # 6
print(b)  # 2
print(c)  # 3 (변경된 적 없음)
```

**출력 결과:**

```
6
2
3
```

## 시간 / 공간 복잡도

| 구분 | 복잡도 | 이유 |
|---|---|---|
| 시간 | O(1) | 고정된 개수의 대입·산술 연산 |
| 공간 | O(1) | 변수 3개만 사용, 입력 크기와 무관 |

## 노하우 & 주의사항

- `a = c` 이후에 나오는 `a`는 **더 이상 원래 값(1)이 아니다.** 문제를 풀 때 헷갈린다면 표를 그려 한 줄씩 값을 갱신하는 습관을 들이면 실수를 줄일 수 있다.
- `c`는 한 번도 재대입되지 않았으므로 끝까지 3을 유지한다 — 코드에 등장하지 않는 변수는 값이 바뀌지 않는다는 점도 함께 확인하자.
- 여러 변수가 얽힌 문제일수록 **실행 순서대로** 하나씩 손으로 따라가는 것이 가장 확실한 방법이다.
