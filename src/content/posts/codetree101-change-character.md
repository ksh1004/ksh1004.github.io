## 핵심 개념

### 변수 재대입(Reassignment)

같은 변수에 값을 다시 대입하면, 이전에 저장되어 있던 값은 사라지고 새 값으로 **덮어써진다**.

```python
a = 'C'   # a라는 이름의 공간에 'C' 저장
a = 'T'   # 같은 공간에 'T'를 다시 저장 → 'C'는 사라짐
```

메모리 공간의 이름(`a`)은 그대로지만, 그 안에 들어있는 값만 교체되는 것이 핵심이다.

### 문자 리터럴

Python에서 작은따옴표(`'`)나 큰따옴표(`"`)로 감싼 한 글자도 문자열(길이 1인 str)로 취급된다. 별도의 문자(char) 타입은 없다.

---

## 접근 방식

1. `a = 'C'` 실행 → `a`에 `'C'` 저장
2. `a = 'T'` 실행 → `a`의 값이 `'C'`에서 `'T'`로 교체됨 (이전 값은 더 이상 참조되지 않음)
3. `print(a)` 실행 시점의 `a` 값은 가장 마지막에 대입된 `'T'`

따라서 출력은 `T`만 나오고, `C`는 출력되지 않는다.

---

## 풀이 코드

```python
# 1) a에 'C'를 저장
a = 'C'
# 2) 같은 변수 a에 'T'를 다시 저장 → 이전 값 'C'는 덮어써짐
a = 'T'
# 3) 현재 a에 저장된 값(= 'T')을 출력
print(a)
```

**출력 결과:**

```
T
```

---

## 시간 / 공간 복잡도

| 구분 | 복잡도 | 이유 |
|---|---|---|
| 시간 | O(1) | 대입과 출력 모두 상수 시간 연산 |
| 공간 | O(1) | 변수 1개만 사용, 입력 크기와 무관 |

---

## 노하우 & 주의사항

- 재대입 문제에서 흔한 실수는 두 값이 모두 출력된다고 착각하는 것이다. `print()`를 호출하지 않는 한 중간 값은 출력되지 않는다.
- `a = 'C'`와 `a = 'T'` 사이에 `print(a)`가 없으므로, `'C'`가 출력에 등장할 일은 없다.
- 여러 번 재대입해도 마지막 대입 값만 살아남는다는 원칙은 몇 번을 덮어써도 동일하게 적용된다.
